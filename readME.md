# JAMA Article Library (Local Search + Viewer)

This repo contains a small toolchain to do two things:

1. Convert JAMA Network articles into a local library of Markdown files and images  
2. Let the research team search and read that local library in a simple web UI

A ZIP of the generated article folders (the `jama_out/` library) will be included in the repo, or provided alongside it.

---

## Important: Adding new articles is a 2-step process

The Streamlit viewer **does not fetch new articles from the internet**. It only displays what already exists in `jama_out/`.

To get **new articles** into the app, do these steps in order:

1. **Rescrape the article list** to refresh `jama_titles.json` (titles + URLs)
2. **Run the batch scraper** to regenerate/update `jama_out/` (markdown + images + index)

If you skip step 1, the scraper will not know about new articles, and nothing new will appear in the viewer.

---

## What’s in this repo

- `app.py`  
  Streamlit web app for searching and viewing the local Markdown library and images.

- `scrape_titles.py`  
  Selenium script that scrapes the JAMA collection pages and regenerates `jama_titles.json` (titles + URLs).  
  Run this first whenever you want new articles.

- `jama_batch.py`  
  Batch scraper that reads `jama_titles.json` and generates/updates:
  - `jama_out/<article_folder>/<folder_name>.md` **or** `jama_out/<article_folder>/article.md` (depends on your scraper)
  - `jama_out/<article_folder>/images/*`
  - `jama_out/index.json`

- `jama_titles.json`  
  Input list of articles with at least `title` and `url`. This is generated by `scrape_titles.py`.

- `jama_out.zip` (or provided separately)  
  The scraped library output. When unzipped, it should create a folder like:
  - `jama_out/index.json`
  - `jama_out/<many article folders>/*.md`
  - `jama_out/<many article folders>/images/*`

## Quick start (view/search the existing articles)

These steps assume you already have the scraped library (`jama_out.zip` or `jama_out/`).

### 1) Put the article library in the right place

If the repo already contains `jama_out/`, skip this.

If the repo contains `jama_out.zip`:

- Unzip `jama_out.zip` in the project root (the same folder as `app.py`)
- After unzipping, you must have:
  - `jama_out/index.json`
  - `jama_out/<many folders>/*.md`
  - `jama_out/<many folders>/images/*`

Expected layout:

```text
project_root/
  app.py
  requirements.txt
  jama_out/
    index.json
    Some_Article_Title__abcdef123456/
      Some_Article_Title__abcdef123456.md   # (or article.md depending on your scraper)
      images/
        figure_1.png
        figure_2.png
```

### 2) Create a virtual environment

macOS / Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
```

Windows (PowerShell):

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
```

### 3) Install dependencies

```bash
pip install -r requirements.txt
```

If you don’t have a `requirements.txt` yet, the minimum to run the viewer app is:

```bash
pip install streamlit
```

### 4) Run the app

```bash
streamlit run app.py
```

Streamlit will print a local URL, typically:

- http://localhost:8501

### 5) Use the app

- Root folder should be `jama_out` (default)
- Search with a few keywords (title and full text are indexed)
- Use the Journal dropdown to filter (if implemented)
- Toggle images on/off (if implemented)
- Images should appear inline when the Markdown contains `![...](images/...)`

## Troubleshooting (Viewer App)

### “Could not load library…”
This usually means the app cannot find `jama_out/index.json` or cannot find the markdown paths referenced by the index.

Fix:
- Confirm you have `jama_out/index.json`
- Confirm `jama_out/` is in the project root (same folder as `app.py`)
- If `jama_out` is elsewhere, update the “Root folder” field in the app to that path

### Images not showing
The app expects images in:

```text
<article_folder>/images/
```

Fix:
- Confirm each article folder has an `images/` directory with figure files
- Confirm the Markdown file sits in the same folder as `images/` (sibling directory)
- If Markdown has inline image tags like `![Figure](images/figure_1.png)`, those should render in place

## Updating the library with new articles (Scrape titles + scrape articles)

Only do this if you want to add new articles or regenerate `jama_out/`.

### Step 1: Rescrape titles/URLs (refresh `jama_titles.json`)

This step discovers new articles from the JAMA collection pages and writes them to `jama_titles.json`.

Run:

```bash
python3 scrape_titles.py
```

Output:
- Updates `jama_titles.json`
- Prints: `Wrote <N> articles to jama_titles.json`

Notes:
- Requires Chrome + ChromeDriver
- The script attempts to accept cookies by clicking a **Continue** button if it appears
- The script paginates until it cannot find new titles or a **Next** link

### Step 2: Run the batch scraper (build/update `jama_out/`)

Example:

```bash
python3 jama_batch.py jama_titles.json OUTDIR=jama_out DOWNLOAD_IMAGES=1 RESUME=1
```

Common options:
- `OUTDIR=jama_out`  
  Where to write output (use the same folder name the app expects)
- `DOWNLOAD_IMAGES=1`  
  Downloads images into each article folder under `images/`
- `HEADLESS=0`  
  Opens a real browser window (useful for login/SSO)
- `RESUME=1`  
  Uses existing `jama_out/index.json` and skips URLs already marked `ok=true`
- `LIMIT=50`  
  Only scrape the first N titles
- `RETRIES=3`  
  Retry failed pages a few times

Example non-headless run:

```bash
python3 jama_batch.py jama_titles.json HEADLESS=0 OUTDIR=jama_out DOWNLOAD_IMAGES=1 RESUME=1
```

Output:
- `jama_out/index.json` (run log + metadata)
- A folder per article containing:
  - a markdown file (either `article.md` or `<folder_name>.md` depending on your implementation)
  - `images/figure_*.png` (when `DOWNLOAD_IMAGES=1`)

## GitHub sharing and what to commit

### Recommended files to commit
- `app.py`
- `scrape_titles.py`
- `jama_batch.py` (and any helper scripts)
- `jama_titles.json`
- `README.md`
- `requirements.txt`
- `jama_out.zip` (or `jama_out/` if it is not too large)

### Do not commit
- `.venv/`
- `__pycache__/`
- `*.pyc`
- `.DS_Store`
- `debug_downloaded.html`

Suggested `.gitignore`:

```text
.venv/
__pycache__/
*.pyc
.DS_Store
debug_downloaded.html
```

## Quick Checklist

1. Ensure `jama_out/` exists and contains `index.json`  
   - If provided as `jama_out.zip`, unzip it into the project root

2. Set up environment  
   - `python3 -m venv .venv`  
   - `source .venv/bin/activate`  
   - `pip install -r requirements.txt`

3. Run the viewer  
   - `streamlit run app.py`

4. To add new articles  
   - `python3 scrape_titles.py`  
   - `python3 jama_batch.py jama_titles.json OUTDIR=jama_out DOWNLOAD_IMAGES=1 RESUME=1`

5. Confirm  
   - Search returns results  
   - Selecting an article displays the markdown  
   - Images render inline
